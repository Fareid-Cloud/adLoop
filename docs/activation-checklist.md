# قائمة التفعيل الكاملة — كل حساب/مفتاح محتاجه عشان AdLoop يشتغل فعلياً

**ملاحظة مهمة:** الملف ده بيتحدث أول بأول كل ما تُضاف ميزة محتاجة إعداد جديد،
مش بيتكتب مرة واحدة من الذاكرة. لو لقيت متغيّر بيئة في `.env.example` مش
موجود هنا، قوللي أضيفه.

---

## 1) قاعدة البيانات (أساسية، أول حاجة)
- [ ] حساب Supabase أو Neon (مجاني) → `DATABASE_URL`
- [ ] `npx prisma migrate dev` بعد الاتصال لأول مرة

## 2) الأمان الأساسي (أساسية)
- [ ] `JWT_SECRET` — أي نص عشوائي طويل تولّده بنفسك (مثلاً `openssl rand -base64 32`)
- [ ] `CRON_SECRET` — نفس الفكرة، نص عشوائي طويل
- [ ] **`TOKEN_ENCRYPTION_KEY` — حرج جداً، مش اختياري:** توكنات جوجل/ميتا
      (وصول مباشر لحسابات إعلانات العملاء) بتتشفّر بيه قبل التخزين. من
      غيره الكود هيرمي خطأ عند أي محاولة ربط حساب. ولّده بـ:
      `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
      **احفظه في مكان آمن منفصل تماماً عن قاعدة البيانات - لو ضاع، كل
      حسابات جوجل/ميتا المربوطة لازم تتربط تاني من الصفر.**

## 3) Google Ads (محتاجة موافقة من جوجل — مش فورية)
- [ ] حساب Google Cloud Console → إنشاء مشروع → تفعيل Google Ads API
- [ ] شاشة موافقة OAuth (OAuth Consent Screen) → `GOOGLE_ADS_CLIENT_ID` + `GOOGLE_ADS_CLIENT_SECRET`
- [ ] **Developer Token من Google Ads نفسه** — ده اللي محتاج وقت (مراجعة من جوجل، مش فوري) → `GOOGLE_ADS_DEVELOPER_TOKEN`
      - Basic Access كافي للبداية (يغطي حسابك انت)، Standard Access لو هتربط عملاء تانيين
- [ ] `redirect_uri` بتاع الـ OAuth لازم يتسجل بالظبط في Google Cloud Console: `{APP_URL}/api/oauth/google-ads/callback`

## 4) Meta Ads (محتاجة موافقة أوسع)
- [ ] حساب Meta for Developers → إنشاء App → `META_APP_ID` + `META_APP_SECRET`
- [ ] تفعيل Marketing API كـ Product على نفس الـ App
- [ ] **App Review من ميتا** — مطلوب قبل ما تقدر تربط حسابات عملاء تانيين غيرك (حسابك انت الشخصي شغال من غيره)
- [ ] `redirect_uri`: `{APP_URL}/api/oauth/meta/callback`

## 4ب) ميتا - جودة ليدز ماسنجر (تكامل منفصل، صلاحيات مختلفة)
- [ ] صلاحيات إضافية على نفس الـ App: `pages_messaging`, `pages_read_engagement`, `pages_manage_metadata`
- [ ] **مراجعة App Review منفصلة محتملة** — `pages_messaging` صلاحية حساسة (قراءة رسايل)، أصعب موافقة من صلاحيات الإعلانات العادية
- [ ] محتاج ربط "Page Access Token" بعد كده (مش نفس توكن حساب الإعلانات) - خطوة تقنية إضافية في الكود لسه هتُبنى

## 4ج) ميتا - معرفة مكان الكليك الحقيقي (فيسبوك/إنستجرام) لطبقة الحقيقة
- [ ] **خطوة يدوية في كل إعلان ميتا** (مش كود، إعداد في Ads Manager نفسه):
      في حقل "URL Parameters" بمستوى الإعلان، ضيف: `site_source_name={{site_source_name}}`
      **من غير الخطوة دي، طبقة الحقيقة (التحويلات المتحقق منها) مش هتعرف
      المكان الأصلي للكليك أبداً، حتى لو الأرقام المُعلنة بتوريه.**
      ده لازم يتضاف لكل إعلان جديد يتعمل، مش مرة واحدة بس.

## 4ز) نظام الدفع (Paymob) - خطوات تفعيل يدوية إجبارية
- [ ] حساب Paymob مصري (accept.paymob.com) - محتاج توثيق أعمال (بياخد لحد 3 أيام حسب توثيقهم)
- [ ] `PAYMOB_SECRET_KEY` و`PAYMOB_PUBLIC_KEY` من Settings في لوحة التحكم
- [ ] `PAYMOB_INTEGRATION_ID` - معرف طريقة الدفع (كارت) من Developers → Payment Integrations
- [ ] **حرج - لازم تتأكد منه بنفسك:** `PAYMOB_HMAC_SECRET` من Settings →
      Payment Integrations → HMAC، **وترتيب الحقول المستخدم في حساب
      التوقيع** (`lib` الكود عندنا فيه تخمين مبني على نمط توثيق قديم،
      مش مؤكد 100% - راجع `app/api/webhooks/paymob/route.ts` وعدّل
      `HMAC_FIELD_ORDER` لو لقيت الترتيب الرسمي مختلف)
- [ ] **يحتاج تفعيل خاص:** الاشتراكات المتكررة عند Paymob محتاجة
      "Moto Integration ID" منفصل عن التكامل العادي - كلّم دعم Paymob
      لتفعيله، مش إعداد ذاتي من لوحة التحكم
- [ ] `PLAN_PRICE_STARTER_CENTS` و`PLAN_PRICE_PRO_CENTS` بالقرش (500 جنيه = 50000)
- [ ] رابط الويب هوك في Paymob → Developers → Payment Integrations →
      Transaction Processed Callback: `https://yourdomain.com/api/webhooks/paymob`
- [ ] **غير مؤكد - يحتاج بحث إضافي:** هل Paymob عندها بوابة عميل ذاتية
      الخدمة (إلغاء اشتراك بنفسه) زي Stripe؟ لو موجودة، `BillingClient.tsx`
      محتاج تحديث لاستخدامها بدل رسالة "تواصل معنا" الحالية

## 4و) تيك توك - أثقل بوابة موافقة من كل المنصات (موثّق ومؤكد)
- [ ] إنشاء تطبيق في TikTok for Business (مش TikTok for Developers العادي
      - نظامين مختلفين تماماً، اتأكدنا منها بالبحث)
- [ ] `TIKTOK_APP_ID` و`TIKTOK_APP_SECRET`
- [ ] Business Center onboarding
- [ ] مراجعة تدريجية (Sandbox → Production) - مش مراجعة تطبيق عادية
- [ ] Business Verification لحجم استخدام أعلى
- [ ] **تدقيق أمان بيانات مخصص (Data Security Compliance Audit)** - أشد

- [ ] ملاحظة إيجابية: التوكن **مبينتهيش افتراضياً** (عكس جوجل/ميتا) -
      مش هتحتاج تجديد دوري، بس محتاج تتأكد منه فعلياً بعد أول ربط حقيقي

## 4هـ) فورم الليدز داخل ميتا (Instant Forms) - محتاج صلاحيات إضافية
- [ ] صلاحيات `leads_retrieval`, `pages_show_list`, `pages_manage_ads` من
      نفس تطبيق ميتا - محتاجة مراجعة App Review زي pages_messaging تماماً
- [ ] اشتراك الـ Page في حقل `leadgen` عن طريق `/subscribed_apps`
- [ ] `META_APP_SECRET` و`META_WEBHOOK_VERIFY_TOKEN` في متغيرات البيئة
- [ ] رابط الويب هوك: `https://yourdomain.com/api/webhooks/meta-leadgen`

## 4د) ميتا/تيك توك - رفع التحويل رجوعاً للمنصة نفسها (لسه مش مبني)
- [ ] `sendMetaConversion` عبر Conversions API - مش موجودة لسه (جوجل بس عندها الدالة دي)
- [ ] `sendTikTokConversion` عبر Events API - مش موجودة لسه

## 4هـ) ميتا - ترقية نوع التوكن (بحث لاحق، مش الآن)
- [ ] **تصحيح مهم اتضح من البحث:** التوكن الحالي (Long-Lived User Token)
      بينتهي كل 60 يوم ومحتاج إعادة موافقة يدوية من العميل. البديل الصح
      لمنصة SaaS بتخدم عملاء متعددين (زينا) مش "System User Token" العادي
      (ده لشركة واحدة بتدير حساباتها هي)، لكن **"Business Integration
      System User" (BISU)** - محتاج تنفيذ "Embedded Signup" (نظام مختلف
      عن OAuth العادي) وغالباً موافقة إضافية كـ Meta Tech Provider.
      **ده تكامل كبير منفصل، مش تعديل سريع - يستاهل تقييم مخصص لاحقاً،
      مش دلوقتي.** الحل المؤقت: تنبيه استباقي قبل انتهاء التوكن (مبني).

## 15) لوحة تحكم المالك (Admin Panel)
- [ ] **خطوة يدوية لمرة واحدة:** بعد أول تسجيل حساب، لازم تحدّث `isAdmin = true`
      على حسابك انت مباشرة في قاعدة البيانات (مفيش UI لعمل admin تاني -
      قرار متعمد، أمان أعلى) - مثال: `UPDATE "User" SET "isAdmin" = true WHERE email = 'your@email.com';`
      بعدها هتقدر توصل لـ `/admin` وتشوف كل المشتركين والحسابات والاستهلاك والمشاكل

## 16) ⏰ عاجل بالتوقيت — Content API لـ Shopping بيتقفل 18 أغسطس 2026

## 14) قانوني (مش كود — لازم محامي متخصص، مش برمجة)
- [ ] **📌 تذكير صريح (بعد ما نخلص موضوع الاختراق/الأمان بس، مش دلوقتي):**
      بانر موافقة الكوكيز (Cookie Consent) يظهر من تحت الشاشة - مطلوب
      قانونياً (قانون حماية البيانات المصري + أي عميل أوروبي محتمل لاحقاً).
      **وأي حاجة شبيهة تظهر أثناء العمل (سياسة خصوصية، شروط استخدام
      كنص فعلي مش بس بند في القائمة، إلخ) هتتسجل هنا وتتنفذ بعد الأمان.**
- [ ] **🔴 عاجل - قانون حماية البيانات المصري (PDPL 151/2020) — موعد نهائي 31 أكتوبر 2026:**
      اللائحة التنفيذية صدرت فعلياً في نوفمبر 2025، والقانون بقى شغّال
      بالكامل. **محتاجين ترخيص من "مركز حماية البيانات الشخصية" (PDPC)**
      لمعظم أنشطة معالجة البيانات، **ترخيص منفصل لنقل البيانات عبر الحدود**
      (وده بيخصنا مباشرة — بياناتنا متخزنة على سيرفرات خارج مصر عبر Vercel/
      قاعدة البيانات)، **تعيين مسؤول حماية بيانات (DPO)** في حالات معينة،
      و**إخطار إجباري خلال 72 ساعة لأي اختراق بيانات**. الوقت المتبقي من
      تاريخ اليوم (18 يوليو 2026) لحد الموعد النهائي: **أقل من 3.5 شهر.**
      **ده أولوية قانونية عاجلة، مش بند عادي في القائمة.**
- [ ] تسجيل العلامة التجارية (اسم AdLoop) - محامي ملكية فكرية
- [ ] شروط الاستخدام وسياسة الخصوصية (Terms of Service / Privacy Policy) - أساسية قبل أي عميل حقيقي، خصوصاً إنك بتتعامل مع بيانات عملاء حساسة
- [ ] اتفاقية عدم إفصاح (NDA) لأي حد هيشتغل معاك على الكود لاحقاً
- [ ] تأمين مسؤولية سيبراني (Cyber Liability Insurance) - حماية مالية لو حصل اختراق رغم كل الإجراءات التقنية

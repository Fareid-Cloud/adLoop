# خريطة استهلاك Claude API — كل نقطة استدعاء في المشروع

**الغرض:** لما نيجي نحسب التكلفة الفعلية أو نوزّعها/نحد منها لاحقاً، دي كل
نقطة بتستهلك من Claude، بمكانها ومتى بتتفعل وبأي حجم تقريبي - بدل ما ندوّر
عليهم واحدة واحدة وقتها.

---

## 1) التحليلات اليومية (`lib/aiInsights.ts` → `generateInsights`)
- **بتتفعل:** تلقائياً مرة واحدة يومياً لكل Workspace فيه `enableAIInsights = true`، عن طريق الـ Cron (`runDailyDiagnosticsForWorkspace`)
- **زائد:** طلب يدوي من المستخدم لو ضغط "تحديث التحليلات" (له نظام Rate Limit منفصل بالفعل — `lib/aiRateLimit.ts`)
- **الحد الحالي المطبّق فعلياً:** 120 طلب يدوي/شهر (~4/يوم) + 30 تلقائي/شهر = 150 إجمالي، وحد ساعي 2 طلب/ساعة — **على مستوى المستخدم مش الـ Workspace** (لو عنده كذا Workspace، الحد مشترك بينهم كلهم)
- **الحجم:** `max_tokens: 1000` لكل استدعاء
- **دالة تانية في نفس الملف:** `detectCreativeFatigue` — **مبنية بس مش مفعّلة فعلياً في أي مكان** (من نتائج المراجعة الشاملة السابقة)

## 2) فحص جودة صور الإعلانات (`lib/imageQualityAudit.ts` → `auditAdImageQuality`)
- **بتتفعل:** عند الطلب بس (زرار "افحص جودة الصورة" في صفحة الإعلانات الفردية) - **مفيش حد أقصى مطبّق حالياً** على عدد المرات اللي ممكن يدوس عليها في اليوم
- **الحجم:** `max_tokens: 800` + صورة كاملة (استهلاك أعلى من النص العادي لأنها Vision)
- ⚠️ **نقطة تستاهل حد لاحقاً** — مفيش Rate Limit عليها زي `aiInsights.ts`، ولو المستخدم عنده حملات كتير وضغط على الزرار في كل واحدة، ممكن يبقى استهلاك عالي بسرعة

## 3) فحص صفحة الهبوط الكامل (`lib/landingPageAudit.ts` + `lib/siteScanOrchestrator.ts`)
- **الحالة الفعلية:** ✅ **شغالة فعلياً دلوقتي** (بعد حل عقبة الصورة عن طريق ScreenshotAPI.net) — مش معلّقة زي ما كانت
- **بتتفعل:** عند الطلب بس (زرار "ابدأ الفحص العميق")، مفيش تلقائي يومي
- **`auditVisualAndCopy`:** `max_tokens: 3000` — استدعاء واحد **لكل رابط** (الرابط الأساسي + لحد منافسين = لحد 3 استدعاءات في الفحص الواحد)
- **`synthesizeAuditReport`:** `max_tokens: 2500` — استدعاء واحد بس للرابط الأساسي (مش بيتكرر للمنافسين)
- **يعني فحص عميق واحد بمنافسين = لحد 4 استدعاءات Claude مجمّعة (~14500 token تقريباً)** — أكبر استهلاك دفعة واحدة في المشروع كله
- ⚠️ **مفيش Rate Limit عليها حالياً** — نفس ملاحظة فحص جودة الصور، تستاهل حد لما نيجي نراجع التكلفة

---

## ملخص الأولويات لما نيجي نحدّ من التكلفة لاحقاً
1. **الأخطر لو اتفعّلت بكثرة:** فحص جودة الصور (رقم 2) — مفيش حد حالياً
2. **الأكبر حجماً لكل استدعاء:** فحص صفحة الهبوط (رقم 3) — لسه مش شغالة عملياً
3. **الأكتر تكراراً لكن محدودة بالفعل:** التحليلات اليومية (رقم 1) — عندها Rate Limit مطبّق من زمان
